# 公共図書館システム ドメインルール

## 1. 概要

公共図書館における **蔵書管理・利用者管理・予約・貸出** を扱うシステムである。

主要な概念:

| 概念 | 説明 |
|------|------|
| Book | ISBN 単位の書誌情報 |
| Book Instance | Book の物理的な蔵書コピー |
| Patron | 図書館利用者 |
| Hold | 予約 |
| Checkout | 貸出 |

---

## 2. Book（書誌情報）

- ISBN を持つ
- タイトルは必須
- 価格は必須
- 図書館カタログに登録される

Book は「概念としての本」であり、物理的な個体ではない。

---

## 3. Book Instance（蔵書コピー）

- ある Book（ISBN）に紐づく
- 登録時に以下のいずれかの種別を決定する

| 種別 | 説明 |
|------|------|
| Circulating | 通常貸出用 |
| Restricted | 研究専用。Researcher のみ予約可能 |

同じ ISBN の Book に対して、Circulating と Restricted の両方の Instance を持てる。

例: 通常版 → Circulating、著者サイン本 → Restricted

---

## 4. Patron（利用者）

利用者には 2 種類ある。

### 4.1 Regular（一般利用者）

- 同時予約は **最大 5 件**
- **無期限予約はできない**
- Restricted 本は予約できない

### 4.2 Researcher（研究者）

- 予約数は **無制限**
- **無期限予約が可能**
- **Restricted 本も予約可能**

---

## 5. Hold（予約）

### 5.1 基本ルール

- ある Book Instance に対して、同時に予約できるのは **1 人だけ**

### 5.2 Hold の種類

| 種類 | 有効期限 | 利用可能な Patron |
|------|----------|-------------------|
| Open-ended（無期限） | チェックアウトされるまで有効 | Researcher のみ |
| Closed-ended（期限付き） | 指定日数以内にチェックアウトされなければ失効 | Regular / Researcher |

### 5.3 期限切れ処理

- Closed-ended Hold は、毎朝の日次処理で期限切れ一覧を確認し、失効処理を行う

---

## 6. Checkout（貸出）

### 6.1 貸出期間

- 最大 **60 日間**

### 6.2 延滞チェック

- **日次** で延滞チェックを実施する

---

## 7. 延滞による制限

- ある支店において、延滞が **2 件を超えている** 利用者は、**その支店での予約が拒否** される

---

## 8. 利用者画面（Patron Profile）

### 8.1 参照できる情報

- 現在の予約（未キャンセル・未失効のもの）
- 現在の貸出（延滞中のものを含む）

### 8.2 実行できる操作

- 本を予約する
- 予約をキャンセルする

---

## 9. カタログ

- 利用者は図書館カタログから本を検索する
- カタログには ISBN 単位で Book が登録されている
- 各 Book に Book Instance が紐づく

---

## 10. ドメイン制約一覧

| # | ルール | 内容 |
|---|--------|------|
| C1 | 同時予約制限 | 1 つの Book Instance に対して同時に予約できるのは 1 人まで |
| C2 | Regular 予約上限 | Regular Patron の同時予約は最大 5 件 |
| C3 | Restricted 制限 | Restricted 本を予約できるのは Researcher のみ |
| C4 | 無期限予約制限 | Open-ended Hold を作成できるのは Researcher のみ |
| C5 | 延滞制限 | 同一支店で延滞 2 件超の Patron は予約拒否 |
| C6 | 貸出期間上限 | 貸出は最大 60 日間 |

---

## 11. 設計検討メモ

ドメインモデルの中心となる概念と、Aggregate 境界の検討ポイント:

- **Book と Book Instance の関係**: Book は Book Instance を持つ集約か？
- **Hold の所属**: Hold は Book Instance に属するか、Patron に属するか？
- **Patron の独立性**: Patron は独立した集約か？
- **永続化の基準**: どの単位でトランザクション整合性を保証するか？
